# 货件 PDF 文件名规范

## 目标

后续批量任务不能依赖已人工整理过的原文件名。系统必须读取 PDF 内容，提取稳定字段，再生成规范文件名。

## 推荐文件名

```text
SKU-产品名-站点-箱数箱[-总件数个]-仓库-FBA物流编码.pdf
```

示例：

```text
3006502-达利白色-澳洲站-25箱-BWU2-FBA15GCL9X61.pdf
1004201-BKYUS-吉米烧火-US-3箱-144个-SWF2-FBA19FRGM282.pdf
1017201-哈兰达-US-7箱-56个-TEB9-FBA19FY790RB.pdf
```

`总件数个` 只有在总件数和箱数不同的时候加入。例如每箱 1 个且 25 箱时，不重复写 `25个`；每箱 48 个且 3 箱时，写 `144个`。

## 字段来源

| 文件名要素 | PDF 内来源 | 示例 |
|---|---|---|
| SKU | `Single SKU` 下方的独立 SKU 行 | `3006502`、`1004201-BKYUS` |
| 产品名 | 优先取黑条标题行 `Created:` 前、仓库前的名称部分；若标题行是 `FBA STA (...)`，回退到 `数量 n` 下一行 | `达利白色25个-BWU2 Created...` → `达利白色`；`数量 1` 下一行 `达利白` |
| 站点 | 目的地地址块里的国家，再映射为命名用站点 | `澳大利亚` → `澳洲站`；`美国` → `US` |
| 箱数 | PDF 页数，同时用 FBA 箱码数量校验 | 25 页 → `25箱` |
| 每箱数量 | 每页右下区域的 `数量 n` | `数量 48` |
| 总件数 | `每箱数量 × PDF 页数` | 48 × 3 → `144个` |
| 仓库 | 黑条标题行中的 `-仓库 Created:`，必要时从目的地地址附近回退识别 | `-BWU2 Created:` → `BWU2` |
| FBA物流编码 | 每页条码下方 `FBA...U000001` 的前 12 位 | `FBA15GCL9X61U000010` → `FBA15GCL9X61` |
| 创建时间 | `Created:` 后文本，仅作审计字段，默认不写入文件名 | `2026/06/10 16:10 AEST (+10)` |

## 不再使用的旧文件名要素

旧文件名末尾常见的长数字，例如：

```text
1781074693949
1781172295750
```

当前样例 PDF 文本层内没有这些数字。它们更像下载流水号或人工处理附加信息，不适合作为自动命名必需字段。后续自动命名默认不保留。

## 产品名例外

部分旧文件名包含比 PDF 更完整的商品称呼：

| SKU | PDF 可见产品名 | 旧文件名产品名 |
|---|---|---|
| `1011902` | `埃尔` | `埃尔收纳架窄款` |
| `1016301` | `盖瑞` | `盖瑞收纳架` |

这类完整品名无法单靠 PDF 内容稳定推断。若后续需要完全还原人工命名，应增加一个 SKU 产品字典，例如：

```json
{
  "1011902": "埃尔收纳架窄款",
  "1016301": "盖瑞收纳架"
}
```

系统命名优先级建议为：

1. SKU 产品字典中的标准品名
2. PDF 黑条标题行清洗后的产品名
3. `数量 n` 下一行的产品简称

## 校验规则

自动重命名前必须满足：

- 能识别 `Single SKU`
- 能识别 SKU
- 能识别产品名
- 能识别目的地国家/站点
- 能识别仓库
- 能识别 FBA 物流编码
- PDF 页数 = 唯一 FBA 箱码数量
- 若 PDF 标题声明 `共 n 个纸箱`，则 `n = PDF 页数`

任一字段缺失或数量不一致时，该文件只进入“需复核”，不自动重命名。
